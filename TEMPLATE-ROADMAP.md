### 발견 항목 상세

#### GAP-001 — PRD 크기 자동 모니터링 + 분할 가이드
**문제**:
- 프로젝트 운영 6개월+ 시 PRD가 1000줄 이상으로 자라남
- AI 에이전트가 PRD 전체를 컨텍스트에 매번 로드 → 토큰 낭비
- 현재 분할 가이드라인이 없음

**해결**:
- [ ] `docs/prd-management.md` 신규 작성 — 분할 시점/방법 명시
- [ ] `stop-doc-checker.sh` 훅에 PRD.md 줄 수 측정 + 1000줄 초과 시 경고 출력
- [ ] `CLAUDE.md`에 PRD 관리 원칙 추가

**검증**:
- 1500줄짜리 더미 PRD를 만든 후 hook이 경고를 정확히 출력하는지

#### GAP-002 — `prd-organizer` 에이전트
**문제**:
- PRD가 커져도 분할 작업 자체는 수동
- 사람이 어떻게 나눠야 할지 판단 부담

**해결**:
- [ ] `.claude/agents/prd-organizer.md` 신규 (Opus 모델)
- [ ] 기능: PRD를 영역별로 자동 분석 → `docs/prd/feature-*.md`로 분할 → `PRD.md`는 인덱스로 재구성
- [ ] `prd-to-roadmap` 에이전트가 분할된 PRD를 인식하도록 수정

#### GAP-003 — 회사 환경 검증
**문제**:
- v0.5는 단위 테스트 + 시나리오 검증만 거침
- 실제 회사 토이 프로젝트로 처음~끝까지 한 번 돌려보지 않음

**해결**:
- [ ] 토이 프로젝트 1개 선정 (예: 사내 부서별 처방 알람 미니 앱)
- [ ] PRD → sprint5까지 완료 + 배포까지 수행
- [ ] 발견된 문제를 v1.0 백로그에 추가

#### GAP-004 — 회고 자동 누적
**현재**: 각 스프린트 회고가 `docs/sprint-retrospectives/sprintN.md`로 분산
**개선**: 패턴 분석 가능하도록 누적 DB(SQLite 또는 단일 JSON) 형태로도 저장

#### GAP-005 — 다국어 지원
**현재**: 한국어 only
**개선**: 영문 README, 영문 CLAUDE.md — 외부 공개 시 유용

## 작업 시작 시 체크리스트

새 GAP 항목 작업 시작:
1. ID 부여 (GAP-XXX 순차)
2. 상태를 `🔄 진행중`으로 변경
3. develop 브랜치에서 작업 시작
4. PR 시 본 ROADMAP의 해당 ID 명시 (예: `feat: PRD 분할 모니터링 (GAP-001)`)
5. 머지 후 상태를 `✅ 완료`로 + 완료일 기록
